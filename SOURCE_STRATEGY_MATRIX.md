# Source Strategy Matrix — Market Intelligence OS v1.0

> *Определить не где больше объявлений, а где больше доказательств личности продавца.*

---

## 1. Purpose

### 1.1 Зачем этот документ

Не все источники данных одинаково полезны. Один может дать 10 000 объявлений и 0 доказательств. Другой — 50 записей и полную картину рынка.

Этот документ определяет:

- **какие источники подключать в первую очередь**
- **какую ценность каждый несёт для Entity Resolution**
- **какие сигналы можно извлечь из каждого**
- **какой уровень доверия к данным каждого источника**

### 1.2 Критерии ранжирования

```yaml
ranking_criteria:
  entity_resolution_value:    # насколько помогает найти продавца
  data_quality:               # структурированность, полнота, достоверность
  coverage:                   # количество продавцов в источнике
  reliability:                # стабильность работы, вероятность ошибок
  extraction_difficulty:      # сложность получения данных (legal + technical)
  trust_level:                # насколько данным можно доверять
```

## 6. Final Principle

> **Не все объявления одинаково полезны. Самое ценное объявление — то, которое помогает найти продавца.**

Объём данных — это не метрика успеха. Entity Resolution Value — вот что определяет ценность источника.

Лучше 50 объявлений из 5 разных источников, чем 5000 из одного.
